# Release Milestone

این سند مرز ریلیز کوچک بعدی را مشخص می‌کند تا توسعه وارد چرخه بی‌پایان بهبود نشود.

## Milestone فعلی: `v0.1-basalam-feedback`

هدف این milestone این است که محصول برای گرفتن بازخورد واقعی از چند فروشنده آماده باشد، نه اینکه همه ایده‌های آینده را کامل کند.

### داخل scope

- مسیر باسلام:
  - انتخاب مسیر باسلام از صفحه اول
  - اتصال غرفه هر فروشنده به غرفه خودش با OAuth
  - حفظ مسیر باسلام بعد از برگشت OAuth
  - آپلود چند عکس با شماره ثابت و resize عکس‌های بزرگ
  - ضبط صدای اختیاری
  - ضبط سازگار با فرمت واقعی Safari/Chrome و وضعیت واضح آماده‌شدن صدا
  - ساخت لیست محصول با AI
  - retry خودکار خطاهای موقت و خروجی نامعتبر AI همراه با log قابل ردیابی
  - بازبینی و ویرایش نام، توضیحات، قیمت، موجودی، زمان آماده‌سازی، وزن، وزن با بسته‌بندی و تعداد هر فروش
  - پیشنهاد دسته‌بندی باسلام و انتخاب دستی اگر لازم شد
  - retry خودکار دسته‌بندی auto اگر باسلام آن را موقع ساخت محصول رد کرد
  - ثبت محصول‌ها در غرفه باسلام و نمایش نتیجه فارسی

- مسیر ترب:
  - انتخاب مسیر ترب از صفحه اول
  - گرفتن اسم فروشگاه و شماره تماس
  - آپلود عکس و ضبط صدای اختیاری
  - ساخت لیست محصول قابل ویرایش
  - ثبت درخواست ترب برای بررسی ادمین
  - جداسازی کامل workspace ترب و باسلام در مرورگر

- تجربه کاربری عمومی:
  - متن‌های اصلی فارسی و غیرفنی
  - عدم نمایش خطای خام انگلیسی به کاربر
  - loading مینیمال برای کارهای زمان‌بر
  - UI قابل استفاده در موبایل و دسکتاپ

### خارج از scope این milestone

- publish مستقیم ترب با سرچ تصویری/متنی واقعی
- ویدئو به‌عنوان ورودی محصول
- تشخیص چند محصول از یک عکس قفسه‌ای با دقت production
- attributeهای اجباری تخصصی همه دسته‌های باسلام
- چند ویس همزمان و مدیریت timeline صدا
- login واقعی، پنل چندکاربره یا سطح دسترسی پیشرفته
- analytics عمیق‌تر از Clarity

## Go / No-Go

برای release این milestone، همه موارد زیر باید پاس باشند:

- `frontend`: `npm test -- --run`
- `frontend`: `npm run build`
- `frontend`: `npm run e2e`
- `backend`: `python -m pytest`
- `git diff --check` بدون خطای whitespace
- health لوکال frontend و backend با status `200`
- smoke دستی باسلام با یک غرفه تست:
  - اتصال غرفه انجام شود و نام غرفه درست نمایش داده شود.
  - دو عکس آپلود شود و شماره‌ها جابه‌جا نشوند.
  - یک ویس کوتاه ضبط شود.
  - لیست ساخته شود.
  - یک محصول ناقص قبل از ثبت، پیام تکمیل اطلاعات بدهد.
  - بعد از تکمیل فیلدها، publish یا موفق شود یا خطای فارسی قابل اقدام بدهد.

No-Go یعنی ریلیز نکنیم اگر یکی از این‌ها رخ داد:

- کاربر بعد از OAuth از مسیر باسلام بیرون بیفتد.
- کاربر دیگری به غرفه اشتباه وصل شود.
- عکس‌های مسیر ترب و باسلام با هم قاطی شوند.
- خطای خام انگلیسی در مسیر اصلی کاربر نمایش داده شود.
- عکس بزرگ UI را از کادر خارج کند یا صفحه را بشکند.
- ثبت باسلام بدون فیلدهای ضروری شروع شود.

## قاعده iteration بعدی

بعد از این milestone، هر iteration باید یکی از این دو شکل باشد:

- bugfix کوچک با تست مرتبط و release سریع
- feature کوچک با scope یک‌خطی، acceptance مشخص، و تصمیم صریح درباره خارج از scope

اگر کاری بیشتر از یک روز توسعه می‌خواهد، قبل از شروع باید milestone جدا بگیرد.
